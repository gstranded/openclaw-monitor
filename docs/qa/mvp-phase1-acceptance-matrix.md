## 验收标准（Checklist）

### A. 监控总览

- [ ] 页面首次加载后，四个核心区域均可见：Agent 状态卡片、积分排行榜、活动时间线、实时事件流
- [ ] 加载成功时，每个区域都能展示最近一次有效数据，不出现误导性的“全部正常”假象
- [ ] 数据为空时，页面展示明确空态文案，而不是报错或空白
- [ ] 局部接口失败时，仅对应模块显示失败/降级状态，其他模块仍可正常渲染
- [ ] 后端不可达时，页面能给出可理解的错误提示与重试入口（若产品定义有重试）
- [ ] 数据延迟或未刷新时，界面能暴露“最后更新时间 / stale 状态”，避免假实时

### B. Agent 状态卡片

- [ ] 能展示 Agent 基础状态（在线/离线/异常/未知）
- [ ] 状态切换后，卡片能在可接受时间内更新
- [ ] 状态未知或字段缺失时，显示兜底值，不出现崩溃或错位
- [ ] 单个 Agent 数据异常不影响其他 Agent 卡片展示

### C. 积分排行榜

- [ ] 能按预期排序展示积分信息
- [ ] 并列分数、缺失分数、负分/异常分值有稳定展示策略
- [ ] 空榜单时有空态提示
- [ ] 局部数据损坏时不会导致整个榜单不可用

### D. 活动时间线

- [ ] 能按时间顺序展示活动记录
- [ ] 无活动时有空态提示
- [ ] 时间字段缺失/格式异常时，界面有兜底而非直接报错
- [ ] 延迟到达的事件不会让时间线顺序明显错误或无提示重排

### E. 实时事件流

- [ ] 新事件能追加到事件流
- [ ] 连接中断/订阅失败时有明确状态提示
- [ ] 重连成功后事件流恢复，不产生明显重复或静默丢失提示缺失
- [ ] 高速事件输入下，界面仍可读，不出现明显卡死或无限抖动

### F. md 文件查看/编辑/保存/同步

- [ ] 能打开关键 md 文件并展示当前内容
- [ ] 编辑后可保存，保存成功有明确反馈
- [ ] 保存失败时保留用户未提交内容，并给出失败原因或可重试提示
- [ ] 外部已更新同一文件时，系统能识别同步冲突或版本变化
- [ ] 同步成功后，查看态与数据源一致
- [ ] 同步失败/冲突时，不应静默覆盖已有内容
- [ ] 空文件、新文件、超长文件、特殊字符/中英文混排都可稳定处理
