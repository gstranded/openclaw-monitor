# Agent 详情页与 Markdown 浏览/编辑实现切片

> 对应任务：Issue #25
> 目的：在正式前端工程落地前，把实现工作拆成可直接认领、可验证、可回归的切片。

## Slice A：页面骨架与 mock 状态

### 目标

先让页面结构、左右栏布局、浏览/编辑态切换在 mock 数据下跑通。

### 范围

- Agent 详情页容器
- Agent 基础信息头部
- 左栏 Markdown 文件列表
- 右栏浏览态 / 编辑态切换
- mock 数据覆盖 editable / readonly / missing

### 完成标准

- 默认打开 `TASK.md`
- `TASK.md` 可进入编辑态
- `AGENTS.md` 只读展示
- 缺失文件能显示 missing 提示

### 建议验证

- 手动切换 3 个文件观察右栏状态变化
- 验证浏览态与编辑态切换无明显状态错乱

## Slice B：真实读取接口接入

### 目标

把 Agent 概览、文件列表、文件内容从 mock 切到真实接口。

### 范围

- 接入 agent 详情接口
- 接入 markdown 文件列表接口
- 接入 markdown 文件内容接口
- 区分整页失败 / 列表失败 / 内容失败

### 完成标准

- 首屏自动加载 agent 信息与文件列表
- 文件切换后内容正确刷新
- 加载态、空态、错误态可区分

### 建议验证

- 接口成功时默认文件选中正确
- 文件内容请求失败时不影响左栏继续切换

## Slice C：编辑与保存链路

### 目标

完成最小可用的单文件编辑、保存、取消和未保存保护。

### 范围

- 文本编辑
- 保存按钮 loading
- 取消编辑
- 未保存切换确认
- 保存成功后刷新版本信息

### 完成标准

- 至少 `TASK.md` 支持完整编辑保存演示
- 保存失败后本地内容保留
- 切换文件时未保存变更会拦截

### 建议验证

- 成功保存后版本或修改时间更新
- 保存失败后输入内容不丢失

## Slice D：边界状态补齐

### 目标

补齐只读、冲突、工作区只读、缺失文件等边界处理。

### 范围

- `editable=false` 的只读说明
- `conflict` 错误提示
- `workspace_readonly` 错误提示
- 列表保留缺失文件位置

### 完成标准

- 用户能明确知道“为什么不能编辑”
- 冲突提示后有明确刷新重试路径
- 缺失文件不从列表中静默消失

### 建议验证

- 手动模拟冲突 / 只读 / 缺失三类场景
- 检查文案是否足够清楚

## 建议交付顺序

1. 先完成 Slice A，尽早给产品 / QA 看页面结构
2. 再完成 Slice B，稳定读取链路
3. 然后完成 Slice C，打通最小保存链路
4. 最后完成 Slice D，补齐边界与回归
